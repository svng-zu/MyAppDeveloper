# PRD: 런닝 앱 개발하려고 하는데,
기본적인 기능 (로그인, 현재 위치, 뛴 시간 , 뛴 거리, 광고 등) 을 포함하고

실시간으로 다른 회원과 1:1 기능을 추가하고싶어. 

# 런닝 앱 PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 1.1 프로젝트명
**RunConnect** (가칭)

### 1.2 프로덕트 비전
혼자 뛰는 것이 지루한 러너들을 위한 실시간 1:1 매칭 기반 소셜 런닝 플랫폼

### 1.3 핵심 가치 제안
- 실시간으로 다른 러너와 1:1 매칭되어 함께 뛸 수 있는 동기부여
- 기본적인 런닝 트래킹과 소셜 기능의 완벽한 결합
- 광고 기반 무료 서비스로 접근성 극대화

## 2. 타겟 사용자 페르소나

### 2.1 Primary Persona: "모티베이션을 찾는 러너 민수"
- **연령**: 25-35세
- **특징**: 런닝을 시작했지만 꾸준히 하기 어려워함
- **Pain Point**: 혼자 뛰기 지루하고, 동기부여 부족
- **Goal**: 재미있게 꾸준히 런닝하고 싶음

### 2.2 Secondary Persona: "경쟁을 즐기는 러너 지영"
- **연령**: 20-40세  
- **특징**: 어느정도 런닝 경험이 있고 경쟁을 통해 성장하고 싶어함
- **Pain Point**: 비슷한 수준의 러닝 파트너를 찾기 어려움
- **Goal**: 실력 향상과 새로운 런닝 경험

## 3. 핵심 기능 (MVP 기준)

### 3.1 기본 기능
- **사용자 인증**: 소셜 로그인 (구글, 카카오, 애플)
- **GPS 런닝 트래킹**: 실시간 위치, 거리, 시간, 페이스 측정
- **런닝 기록 저장**: 개인 런닝 히스토리 관리
- **광고 시스템**: 배너 및 전면 광고

### 3.2 차별화 기능
- **실시간 1:1 매칭**: 비슷한 수준/위치의 러너와 실시간 매칭
- **버츄얼 런닝**: 매칭된 상대와 실시간으로 속도/거리 비교하며 런닝
- **음성 채팅**: 런닝 중 간단한 음성 응원/소통 기능

## 4. 사용자 스토리

### 4.1 기본 런닝 기능
```
As a 러너,
I want to 내 런닝 기록(시간, 거리, 페이스)을 정확히 측정하고 싶다,
So that 내 운동 성과를 객관적으로 파악할 수 있다.
```

### 4.2 실시간 매칭 기능
```
As a 런닝을 시작하는 사용자,
I want to 비슷한 실력의 다른 러너와 실시간으로 매칭되고 싶다,
So that 동기부여를 받으며 더 재미있게 런닝할 수 있다.
```

### 4.3 버츄얼 런닝 기능
```
As a 매칭된 러너,
I want to 상대방의 실시간 런닝 상황(속도, 거리)을 보고 싶다,
So that 경쟁심을 느끼며 더 열심히 뛸 수 있다.
```

## 5. 기술적 요구사항

### 5.1 클라이언트
- **플랫폼**: iOS, Android (React Native 또는 Flutter)
- **GPS**: 고정밀 위치 추적 (1초마다 업데이트)
- **실시간 통신**: WebSocket 기반 실시간 데이터 동기화
- **지도**: Google Maps 또는 Apple Maps SDK

### 5.2 서버
- **Backend**: Node.js + Express 또는 Python + FastAPI
- **Database**: 
  - PostgreSQL (사용자 데이터, 런닝 기록)
  - Redis (실시간 매칭, 세션 관리)
- **실시간 매칭**: 지리적 위치 + 러닝 능력 기반 매칭 알고리즘
- **인프라**: AWS 또는 GCP (Auto Scaling 지원)

### 5.3 외부 연동
- **광고**: Google AdMob
- **소셜 로그인**: OAuth 2.0 (Google, Kakao, Apple)
- **음성 채팅**: Agora.io 또는 WebRTC
- **푸시 알림**: FCM (Firebase Cloud Messaging)

## 6. 개발 우선순위

### 6.1 Phase 1 (MVP - 8주)
1. **사용자 인증 시스템** (1주)
2. **기본 런닝 트래킹** (2주)
3. **런닝 기록 저장/조회** (1주)
4. **기본 UI/UX** (2주)
5. **광고 시스템 통합** (1주)
6. **베타 테스트 및 버그픽스** (1주)

### 6.2 Phase 2 (소셜 기능 - 6주)
1. **실시간 매칭 알고리즘** (2주)
2. **매칭 대기/수락 UI** (1주)
3. **실시간 런닝 데이터 동기화** (2주)
4. **버츄얼 런닝 화면** (1주)

### 6.3 Phase 3 (고도화 - 4주)
1. **음성 채팅 기능** (2주)
2. **성과 분석 및 통계** (1주)
3. **사용자 프로필 및 랭킹** (1주)

### 6.4 성공 지표 (KPI)
- **사용자 확보**: 출시 3개월 내 MAU 10,000명
- **매칭 성공률**: 매칭 요청의 80% 이상 성공
- **사용자 리텐션**: 7일 리텐션 40% 이상
- **수익화**: 광고 기반 ARPU $0.5 이상

이 PRD를 바탕으로 개발팀과 상세 기술 검토 및 일정 조율을 진행하시기 바랍니다. 특히 실시간 매칭 알고리즘의 복잡성을 고려하여 충분한 테스트 기간을 확보하는 것이 중요합니다.
