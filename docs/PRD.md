# PRD: 런닝앱을 만들려고해. 내 런닝앱은 기본적인 MVP인데sns 연동 로그인과 사용자들과 1:1 런닝 배틀을 뜰 수 있는 기능등이 필요해 이를 기반으로 너가 기획해줘.

# 런닝 배틀 앱 PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 프로젝트명: RunBattle
### 비전
러너들이 실시간으로 1:1 배틀을 통해 동기부여를 받고 재미있게 달릴 수 있는 소셜 런닝 플랫폼

### 목표
- 혼자 달리기 지루한 러너들에게 경쟁의 재미 제공
- 실시간 배틀을 통한 운동 동기부여 증진
- 런닝 커뮤니티 형성 및 활성화

### 핵심 가치 제안
- **실시간 경쟁**: 전 세계 러너들과 즉시 매칭되어 배틀 가능
- **공정한 배틀**: 유사한 실력의 상대와 매칭
- **즉시성**: 언제든지 빠른 매칭으로 배틀 시작

## 2. 타겟 사용자 페르소나

### 페르소나 1: 경쟁형 러너 (김민수, 28세)
- **특징**: 주 3-4회 5km 이상 달리는 중급 러너
- **니즈**: 혼자 달리기 지루함, 새로운 동기부여 필요
- **행동**: 런닝 앱으로 기록 관리, SNS에 운동 인증 게시
- **목표**: 개인기록 향상, 타인과의 경쟁을 통한 성취감

### 페르소나 2: 소셜 러너 (이지영, 24세)
- **특징**: 건강 관리 목적의 초보~중급 러너
- **니즈**: 운동 습관 형성, 재미있는 운동 방법 탐색
- **행동**: 친구들과 함께 운동, 운동 관련 콘텐츠 소비
- **목표**: 꾸준한 운동 습관, 새로운 사람들과의 교류

### 페르소나 3: 데이터 중시 러너 (박성호, 35세)
- **특징**: 마라톤 완주 경험 있는 상급 러너
- **니즈**: 정확한 데이터 기반 훈련, 성과 측정
- **행동**: 다양한 피트니스 앱 활용, 러닝 기어에 투자
- **목표**: 마라톤 기록 단축, 체계적인 훈련

## 3. 핵심 기능 (MVP 기준)

### 3.1 사용자 관리
- SNS 연동 로그인 (구글, 카카오, 애플)
- 프로필 생성 및 관리
- 런닝 레벨/티어 시스템

### 3.2 런닝 트래킹
- GPS 기반 거리/시간/속도 측정
- 실시간 페이스 모니터링
- 기본 런닝 통계 제공

### 3.3 1:1 배틀 시스템
- 실시간 매칭 시스템
- 레벨/실력 기반 매칭
- 실시간 배틀 진행 화면
- 배틀 결과 및 리워드

### 3.4 소셜 기능
- 친구 시스템 (팔로우/팔로잉)
- 배틀 히스토리 공유
- 간단한 피드 기능

## 4. 사용자 스토리

### 회원가입/로그인
- **As a** 신규 사용자, **I want to** SNS 계정으로 간편하게 가입하고 **so that** 별도 정보 입력 없이 빠르게 서비스를 이용할 수 있다.
- **As a** 기존 사용자, **I want to** 자동 로그인되어 **so that** 매번 로그인 과정 없이 바로 앱을 사용할 수 있다.

### 프로필 설정
- **As a** 사용자, **I want to** 내 런닝 실력과 선호하는 거리를 설정하고 **so that** 적절한 상대와 매칭될 수 있다.
- **As a** 사용자, **I want to** 프로필 사진과 닉네임을 설정하고 **so that** 다른 사용자들과 구별될 수 있다.

### 배틀 매칭
- **As a** 러너, **I want to** '배틀 찾기' 버튼을 누르면 **so that** 비슷한 실력의 상대와 자동으로 매칭될 수 있다.
- **As a** 러너, **I want to** 배틀 거리(1km, 3km, 5km)를 선택하고 **so that** 내가 원하는 조건으로 경기할 수 있다.

### 실시간 배틀
- **As a** 러너, **I want to** 배틀 중 상대방의 실시간 위치와 페이스를 보고 **so that** 경쟁 상황을 파악하며 달릴 수 있다.
- **As a** 러너, **I want to** 배틀 중 포기 옵션이 있어서 **so that** 부상 등의 상황에서 안전하게 중단할 수 있다.

### 결과 및 보상
- **As a** 러너, **I want to** 배틀 종료 후 상세한 결과를 보고 **so that** 내 성과와 상대방과의 차이를 분석할 수 있다.
- **As a** 러너, **I want to** 승리 시 포인트나 배지를 획득하여 **so that** 성취감을 느끼고 지속적인 동기부여를 받을 수 있다.

### 기록 관리
- **As a** 러너, **I want to** 내 모든 런닝 기록을 한 곳에서 보고 **so that** 발전 상황을 추적할 수 있다.
- **As a** 러너, **I want to** 월별/주별 통계를 확인하여 **so that** 운동 패턴을 파악하고 목표를 설정할 수 있다.

## 5. 기술적 요구사항

### 5.1 플랫폼
- iOS (Swift) / Android (Kotlin) 네이티브 앱
- 최소 지원 버전: iOS 13.0+ / Android API 21+

### 5.2 외부 연동
- **SNS 로그인**: Google OAuth, Kakao Login, Apple Sign-in
- **지도/위치**: Google Maps SDK, Apple MapKit
- **실시간 통신**: WebSocket 또는 Firebase Realtime Database
- **푸시 알림**: FCM (Firebase Cloud Messaging)

### 5.3 데이터 요구사항
- GPS 정확도: 5m 이내
- 실시간 데이터 동기화 지연시간: 3초 이내
- 배틀 매칭 시간: 30초 이내

### 5.4 성능 요구사항
- 앱 초기 로딩 시간: 3초 이내
- GPS 신호 수신 시간: 10초 이내
- 배틀 중 데이터 전송 주기: 5초마다

### 5.5 보안 요구사항
- 개인정보 암호화 저장
- HTTPS 통신
- 위치 데이터 보안 처리

## 6. 개발 우선순위

### Phase 1 (MVP - 4주)
1. **사용자 인증**: SNS 로그인, 프로필 생성
2. **기본 런닝 트래킹**: GPS 기반 거리/시간 측정
3. **간단한 배틀 시스템**: 매칭, 실시간 진행, 결과 확인

### Phase 2 (기능 확장 - 3주)
1. **레벨 시스템**: 티어 기반 매칭 개선
2. **소셜 기능**: 친구 추가, 기본 피드
3. **통계 기능**: 개인 기록 관리, 그래프

### Phase 3 (사용자 경험 개선 - 2주)
1. **UI/UX 개선**: 배틀 화면 최적화
2. **알림 시스템**: 매칭 완료, 배틀 결과 푸시
3. **성능 최적화**: 배터리 사용량, 데이터 사용량 최적화

### Phase 4 (추가 기능 - 3주)
1. **보상 시스템**: 포인트, 배지, 리더보드
2. **커뮤니티**: 배틀 히스토리 공유, 댓글
3. **고급 통계**: 상세 분석, 개인 기록 예측

### 개발 완료 기준
- 모든 핵심 기능이 안정적으로 동작
- 사용자 테스트 완료 (최소 50명)
- 앱스토어 심사 통과 준비 완료
- 서버 안정성 테스트 완료
