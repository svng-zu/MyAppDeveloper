# PRD: 런닝앱 개발 (로그인 , 광고 배너, 실제 런닝 기록, 사용자 커뮤니티 공유기능)


# 런닝 앱 PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 1.1 프로덕트 개요
**제품명**: RunTogether (러닝 기록 & 커뮤니티 앱)

**비전**: 러너들이 자신의 러닝 기록을 체계적으로 관리하고, 다른 러너들과 경험을 공유하며 동기부여를 받을 수 있는 종합 러닝 플랫폼

**미션**: 모든 레벨의 러너들이 지속적으로 러닝을 즐길 수 있도록 기록 관리와 커뮤니티 기능을 통해 동기부여와 성취감을 제공

### 1.2 비즈니스 목표
- 월간 활성 사용자(MAU) 10만명 달성 (1년 내)
- 광고 수익을 통한 수익화 모델 구축
- 사용자 리텐션율 60% 이상 유지 (1개월 기준)

## 2. 타겟 사용자 페르소나

### 2.1 주요 페르소나

**페르소나 1: 초보 러너 김민수 (28세, 직장인)**
- 러닝 경력: 6개월 미만
- 주요 니즈: 체계적인 기록 관리, 동기부여, 러닝 정보 습득
- 사용 패턴: 주 2-3회 러닝, 기록 확인 및 커뮤니티 글 읽기

**페르소나 2: 중급 러너 박지영 (35세, 주부)**
- 러닝 경력: 2-3년
- 주요 니즈: 기록 향상 추적, 같은 수준 러너들과의 교류
- 사용 패턴: 주 4-5회 러닝, 적극적인 커뮤니티 참여

**페르소나 3: 마라톤 러너 이성호 (42세, 자영업)**
- 러닝 경력: 5년 이상
- 주요 니즈: 전문적인 기록 분석, 노하우 공유, 러닝 메이트 찾기
- 사용 패턴: 거의 매일 러닝, 커뮤니티 리더 역할

## 3. 핵심 기능 (MVP 기준)

### 3.1 사용자 인증 시스템
- 이메일/전화번호 회원가입 및 로그인
- 소셜 로그인 (Google, Apple, 카카오)
- 프로필 설정 (닉네임, 프로필 사진, 기본 정보)

### 3.2 러닝 기록 관리
- GPS 기반 실시간 러닝 트래킹
- 거리, 시간, 속도, 칼로리 소모량 기록
- 러닝 경로 지도 표시
- 개인 기록 통계 (일/주/월별)

### 3.3 커뮤니티 기능
- 러닝 기록 공유 (피드 형태)
- 게시글 작성/수정/삭제
- 댓글 및 좋아요 기능
- 팔로우/팔로워 시스템

### 3.4 광고 시스템
- 배너 광고 (메인 화면, 피드 중간)
- 전면 광고 (앱 실행 시, 러닝 완료 후)
- 네이티브 광고 (피드 내 통합)

## 4. 사용자 스토리

### 4.1 인증 관련
```
As a 신규 사용자
I want to 간편하게 회원가입을 하고 싶다
So that 빠르게 앱을 사용할 수 있다

As a 기존 사용자  
I want to 자동 로그인이 되길 원한다
So that 매번 로그인 정보를 입력하지 않아도 된다
```

### 4.2 러닝 기록 관련
```
As a 러너
I want to 내 러닝을 정확하게 기록하고 싶다
So that 나의 발전 상황을 확인할 수 있다

As a 러너
I want to 과거 기록들을 쉽게 조회하고 싶다  
So that 나의 러닝 패턴을 분석할 수 있다
```

### 4.3 커뮤니티 관련
```
As a 러너
I want to 내 러닝 성과를 다른 사람들과 공유하고 싶다
So that 동기부여를 받고 격려받을 수 있다

As a 커뮤니티 사용자
I want to 다른 러너들의 글에 반응하고 싶다
So that 서로 소통하고 응원할 수 있다
```

## 5. 기술적 요구사항

### 5.1 플랫폼
- **모바일**: iOS 13.0+, Android 8.0+ (API 26+)
- **개발 방식**: React Native 또는 Flutter (크로스 플랫폼)

### 5.2 백엔드 아키텍처
- **서버**: Node.js + Express 또는 Spring Boot
- **데이터베이스**: PostgreSQL (메인), Redis (캐시)
- **클라우드**: AWS 또는 GCP
- **파일 저장소**: AWS S3 또는 Google Cloud Storage

### 5.3 핵심 기술 스택
- **GPS 및 위치 서비스**: 네이티브 GPS API
- **지도 서비스**: Google Maps API 또는 Naver Maps API
- **실시간 통신**: WebSocket (채팅, 실시간 알림)
- **푸시 알림**: FCM (Firebase Cloud Messaging)
- **광고 SDK**: Google AdMob, Facebook Audience Network

### 5.4 보안 요구사항
- JWT 기반 인증 시스템
- HTTPS 통신 (SSL/TLS)
- 개인정보 암호화 저장
- API Rate Limiting

### 5.5 성능 요구사항
- 앱 로딩 시간: 3초 이내
- GPS 위치 업데이트: 5초 간격
- 피드 로딩: 2초 이내
- 이미지 업로드: 10MB 이하

## 6. 개발 우선순위

### 6.1 Phase 1 (MVP - 8주)
**Week 1-2: 기본 인프라 구축**
- 프로젝트 환경 설정
- 기본 UI/UX 프레임워크 구축
- 사용자 인증 시스템 개발

**Week 3-4: 러닝 기록 기능**
- GPS 트래킹 기능 구현
- 기본 러닝 데이터 수집 및 저장
- 개인 기록 조회 화면

**Week 5-6: 커뮤니티 기본 기능**
- 게시글 작성/조회 기능
- 댓글 및 좋아요 기능
- 기본 피드 구현

**Week 7-8: 광고 및 최종 테스트**
- 광고 SDK 통합
- 배너 광고 구현
- QA 테스트 및 버그 수정

### 6.2 Phase 2 (고도화 - 4주)
- 러닝 통계 및 분석 기능 강화
- 팔로우/팔로워 시스템
- 전면 광고 및 네이티브 광고 추가
- 성능 최적화

### 6.3 Phase 3 (확장 - 6주)
- 러닝 챌린지 기능
- 그룹 러닝 기능
- 상세 러닝 분석 (심박수, 페이스 등)
- 개인화 추천 시스템

### 6.4 성공 지표 (KPI)
- **사용자 획득**: 월 신규 가입자 5,000명
- **참여도**: 일 평균 러닝 기록 500건
- **커뮤니티 활성도**: 일 평균 게시글 100건, 댓글 300건  
- **수익화**: 월 광고 수익 $2,000 이상
- **리텐션**: D7 40%, D30 25%

이 PRD를 바탕으로 개발팀과 함께 상세한 기술 설계와 일정 계획을 수립하시길 권합니다.
